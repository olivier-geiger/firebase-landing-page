01

Console Firebase

Création projet · iOS App · GoogleService-Info.plist

Tout commence sur console.firebase.google.com. Créez le projet Firebase avant de toucher à Xcode — c'est le projet Firebase qui génère le fichier de configuration dont votre app a besoin.

Créer le projet Firebase

1

Nouveau projet

console.firebase.google.com → Créer un projet. Donnez un nom clair (MonApp-Production). Créez également un projet séparé pour le staging : MonApp-Staging. Un projet Firebase par environnement.

⚠ Choisissez le bon emplacement des données (europe-west1 pour RGPD)
2

Ajouter une app iOS

Vue d'ensemble du projet → Ajouter une app → iOS. Renseignez l'identifiant d'app (Bundle ID) exactement comme dans Xcode : com.votresociete.monapp. Facultatif : surnom, App Store ID.

3

Télécharger GoogleService-Info.plist

Firebase génère un fichier GoogleService-Info.plist unique par app iOS. Téléchargez-le et ajoutez-le dans Xcode en le glissant directement sur le target (pas dans un groupe). Cochez "Copy items if needed" et sélectionnez le bon target.

✅ Ce fichier contient l'API_KEY, PROJECT_ID, BUNDLE_ID, APP_ID Firebase
4

Ne jamais commiter GoogleService-Info.plist

Ajoutez GoogleService-Info.plist à votre .gitignore. Distribuez-le via votre CI/CD (GitHub Secrets, Fastlane Match) ou les variables d'environnement. Le fichier contient des clés sensibles lisibles par n'importe qui ayant accès au repo.

⚠️

Plusieurs environments : si vous avez Debug/Staging/Release, vous aurez plusieurs GoogleService-Info.plist. Stockez-les dans des dossiers séparés (Firebase/Debug/, Firebase/Release/) et utilisez un script de build Xcode pour copier le bon fichier selon la configuration active.

Script de build Xcode — copier le bon GoogleService-Info.plist
#!/bin/bash
# Build Phase → Run Script → AVANT "Copy Bundle Resources"

case "${CONFIGURATION}" in
  "Debug")
    cp "${PROJECT_DIR}/Firebase/Debug/GoogleService-Info.plist" \
       "${BUILT_PRODUCTS_DIR}/${PRODUCT_NAME}.app/GoogleService-Info.plist"
    ;;
  "Release")
    cp "${PROJECT_DIR}/Firebase/Release/GoogleService-Info.plist" \
       "${BUILT_PRODUCTS_DIR}/${PRODUCT_NAME}.app/GoogleService-Info.plist"
    ;;
  *)
    echo "Unknown configuration: ${CONFIGURATION}"
    exit 1
    ;;
esac

03

Initialisation Firebase

FirebaseApp.configure · AppDelegate · @main

Firebase doit être initialisé avant tout accès à ses services, le plus tôt possible dans le cycle de vie de l'app. L'erreur la plus fréquente est d'appeler un service Firebase avant FirebaseApp.configure().

MyApp.swift — initialisation Firebase dans @main
import SwiftUI
import Firebase
import FirebaseCore

@main
struct MyApp: App {

    // AppDelegate pour Firebase + push notifications
    @UIApplicationDelegateAdaptor(AppDelegate.self) var appDelegate

    @StateObject private var container = AppContainer.shared

    var body: some Scene {
        WindowGroup {
            AppRootView()
                .environmentObject(container)
                .environmentObject(container.themeManager)
                .preferredColorScheme(container.themeManager.current.colorScheme)
        }
    }
}

class AppDelegate: NSObject, UIApplicationDelegate {

    func application(
        _ application: UIApplication,
        didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
    ) -> Bool {

        // ① Toujours en PREMIER — avant tout autre SDK
        FirebaseApp.configure()

        // ② Crashlytics — active le crash handler
        Crashlytics.crashlytics().setCrashlyticsCollectionEnabled(
            AppConfiguration.crashlyticsEnabled
        )

        // ③ Analytics — désactiver en debug si souhaité
        Analytics.setAnalyticsCollectionEnabled(
            AppConfiguration.analyticsEnabled
        )

        return true
    }
}
✅

AppConfiguration : on réutilise ici l'AppConfiguration défini dans Core — analyticsEnabled retourne false en Debug et true en Release. Crashlytics pareil. Cela évite de polluer vos données Analytics avec vos propres sessions de développement.

04

Sign in with Apple

Console Firebase · Apple Developer · Service ID · Clé privée

Configurer Sign in with Apple avec Firebase nécessite des étapes à la fois sur le portail Apple Developer ET dans la console Firebase. Les deux doivent être synchronisés — c'est la source principale d'erreurs.

Côté Apple Developer

1

Activer Sign in with Apple sur l'App ID

developer.apple.com → Identifiers → votre App ID → cochez "Sign in with Apple" → Save. Cela active la capability côté portail.

2

Créer un Service ID

Identifiers → + → Services IDs. Identifiant : com.votresociete.monapp.siwa (différent du Bundle ID). Activez "Sign in with Apple" → Configure → ajoutez le domaine Firebase : monapp-xxxxx.firebaseapp.com et l'URL de retour : https://monapp-xxxxx.firebaseapp.com/__/auth/handler

Le domaine Firebase se trouve dans Firebase Console → Authentication → Sign-in method → Apple
3

Créer une clé privée

Clés → + → cochez "Sign in with Apple" → Configure → sélectionnez votre App ID principal. Téléchargez le fichier .p8. À faire une seule fois — impossible de retélécharger. Notez le Key ID (10 caractères).

⚠ Stockez le .p8 en lieu sûr — jamais dans le repo git

Côté Firebase Console

4

Configurer le provider Apple

Authentication → Sign-in method → Apple → Activer. Renseignez :

Services ID : com.votresociete.monapp.siwa
5

Renseigner la clé OAuth

Team ID (10 caractères, visible sur developer.apple.com) + Key ID (du fichier .p8) + contenu du fichier .p8. Ces informations permettent à Firebase de valider les tokens Apple côté serveur.

ℹ️

Pourquoi Firebase a besoin du .p8 ? Quand un utilisateur se connecte avec Apple, l'app reçoit un identityToken JWT signé par Apple. Firebase doit vérifier ce token côté serveur en utilisant votre clé privée pour confirmer qu'il est authentique. Sans ça, Firebase ne peut pas créer l'utilisateur Firebase associé.

Data/Auth/AuthService.swift — Sign in with Apple complet
import FirebaseAuth
import AuthenticationServices
import CryptoKit

// MARK: — LAYER: Data
@MainActor
final class AuthService: AuthServiceProtocol, ObservableObject {

    @Published private(set) var currentUser: AppUser?
    private var stateListener: AuthStateDidChangeListenerHandle?
    // Nonce SHA256 — protège contre les replay attacks
    private var currentNonce: String?
    private let keychain: KeychainService

    init(keychain: KeychainService) {
        self.keychain = keychain
        observeAuthState()
    }

    deinit {
        if let h = stateListener { Auth.auth().removeStateDidChangeListener(h) }
    }

    // ── Observer l'état d'authentification ─────────────
    private func observeAuthState() {
        stateListener = Auth.auth().addStateDidChangeListener { [weak self] _, user in
            Task { await MainActor.run {
                self?.currentUser = user.map(AppUser.init)
                if let uid = user?.uid {
                    self?.keychain.set(uid, forKey: .userID)
                }
            } }
        }
    }

    // ── Préparer la requête Apple Sign In ──────────────
    func prepareAppleRequest(_ request: ASAuthorizationAppleIDRequest) {
        let nonce = randomNonceString()
        currentNonce = nonce
        request.requestedScopes = [.fullName, .email]
        request.nonce = sha256(nonce)  // envoyé à Apple sous forme hashée
    }

    // ── Compléter la connexion Apple ───────────────────
    func signInWithApple(authorization: ASAuthorization) async throws -> AppUser {
        guard let appleIDCred = authorization.credential as? ASAuthorizationAppleIDCredential,
              let nonce        = currentNonce,
              let tokenData    = appleIDCred.identityToken,
              let tokenStr     = String(data: tokenData, encoding: .utf8)
        else { throw AuthError.invalidCredential }

        // Construire le credential Firebase depuis le token Apple
        let credential = OAuthProvider.appleCredential(
            withIDToken:    tokenStr,
            rawNonce:       nonce,       // nonce non hashé — Firebase le hashe et compare
            fullName:       appleIDCred.fullName
        )

        let result = try await Auth.auth().signIn(with: credential)

        // Sauvegarder l'identifiant Apple stable dans le Keychain
        if let appleUID = appleIDCred.user {
            keychain.set(appleUID, forKey: .appleUserID)
        }

        let user = AppUser(result.user)
        currentUser = user
        return user
    }

    // ── Déconnexion ────────────────────────────────────
    func signOut() throws {
        try Auth.auth().signOut()
        keychain.clearAll()
        currentUser = nil
    }

    // ── Vérifier la validité du token Apple côté client ─
    func checkAppleCredentialState() async throws -> ASAuthorizationAppleIDProvider.CredentialState {
        guard let appleUID = keychain.get(.appleUserID) else {
            return .notFound
        }
        return try await ASAuthorizationAppleIDProvider().credentialState(forUserID: appleUID)
    }

    // ── Helpers cryptographiques ───────────────────────
    private func randomNonceString(length: Int = 32) -> String {
        let charset = Array("0123456789ABCDEFGHIJKLMNOPQRSTUVXYZabcdefghijklmnopqrstuvwxyz-._")
        var result = ""
        var remainingLength = length
        while remainingLength > 0 {
            let randoms: [UInt8] = (0<16).map { _ in
                var r: UInt8 = 0
                SecRandomCopyBytes(kSecRandomDefault, 1, &r)
                return r
            }
            randoms.forEach { r in
                if remainingLength == 0 { return }
                if r < charset.count { result.append(charset[Int(r)]); remainingLength -= 1 }
            }
        }
        return result
    }

    private func sha256(_ input: String) -> String {
        SHA256.hash(data: Data(input.utf8))
            .compactMap { String(format: "%02x", $0) }
            .joined()
    }
}

        self.email         = user.email

@MainActor
final class SignInViewModel: ObservableObject {
    @Published var isLoading    = false
    @Published var errorMessage: String?
    let authService: AuthService

    func handleAppleResult(_ result: Result<ASAuthorization, Error>) async {
        isLoading = true; errorMessage = nil
        defer { isLoading = false }
        do {
            let auth = try result.get()
            try await authService.signInWithApple(authorization: auth)
        } catch ASAuthorizationError.canceled {
            // Annulation par l'utilisateur — ne pas afficher d'erreur
        } catch {
            errorMessage = error.localizedDescription
        }
    }
}

Firestore — Data Layer

Repository · Codable · Real-time listeners · Collections

Firestore s'intègre dans la couche Data via un repository. Les documents Firestore sont mappés en entités Domain via Codable. Aucune référence à Firestore ne sort de la couche Data.

Structure des collections

firestore/
├── users/                         ← collection
│   └── {uid}/                     ← document (=Firebase UID)
│       ├── displayName: String
│       ├── email: String?
│       ├── subscriptionTier: String
│       ├── createdAt: Timestamp
│       └── settings/             ← sous-collection
│           └── preferences/
│
├── tasks/                         ← collection racine
│   └── {taskID}/
│       ├── ownerUID: String       ← pour filtrage + rules
│       ├── title: String
│       ├── notes: String
│       ├── isDone: Bool
│       ├── priority: Int
│       ├── dueDate: Timestamp?
│       └── createdAt: Timestamp
ℹ️

Collection racine vs sous-collection : les tâches sont dans une collection racine tasks/ avec un champ ownerUID plutôt que dans users/{uid}/tasks/. Cela permet des requêtes globales (admin) et facilite les règles de sécurité. Les deux approches fonctionnent — choisissez selon vos besoins.

Core/Constants/FirestoreConstants.swift — jamais de string collection en dur ailleurs
enum FirestoreConstants {
    enum Collections {
        static let users  = "users"
        static let tasks  = "tasks"
    }
    enum Fields {
        static let ownerUID   = "ownerUID"
        static let createdAt  = "createdAt"
        static let isDone     = "isDone"
    }
}
Data/Firestore/TaskFirestoreRepository.swift
import FirebaseFirestore
import FirebaseAuth

// MARK: — LAYER: Data — FirebaseFirestore importé ici seulement
final class TaskFirestoreRepository: TaskRepositoryProtocol {

    private let db = Firestore.firestore()
    private var listener: ListenerRegistration?

    private var collection: CollectionReference {
        db.collection(FirestoreConstants.Collections.tasks)
    }

    private var currentUID: String? {
        Auth.auth().currentUser?.uid
    }

    // ── Fetch one-shot ─────────────────────────────────
    func fetchAll() async throws -> [Task] {
        guard let uid = currentUID else { throw AuthError.notAuthenticated }

        let snapshot = try await collection
            .whereField(FirestoreConstants.Fields.ownerUID, isEqualTo: uid)
            .order(by: FirestoreConstants.Fields.createdAt, descending: true)
            .getDocuments()

        return snapshot.documents.compactMap { try? $0.data(as: TaskDTO.self).toDomain() }
    }

    // ── Listener temps réel ────────────────────────────
    func observeAll(onChange: @escaping ([Task]) -> Void) {
        guard let uid = currentUID else { return }
        listener = collection
            .whereField(FirestoreConstants.Fields.ownerUID, isEqualTo: uid)
            .order(by: FirestoreConstants.Fields.createdAt, descending: true)
            .addSnapshotListener { snapshot, _ in
                let tasks = snapshot?.documents
                    .compactMap { try? $0.data(as: TaskDTO.self).toDomain() }
                    ?? []
                onChange(tasks)
            }
    }

    func stopObserving() { listener?.remove() }

    // ── Créer ──────────────────────────────────────────
    func create(_ task: Task) async throws {
        guard let uid = currentUID else { throw AuthError.notAuthenticated }
        var dto = TaskDTO(task, ownerUID: uid)
        try collection.document(task.id.uuidString).setData(from: dto)
    }

    // ── Mettre à jour ──────────────────────────────────
    func update(_ task: Task) async throws {
        try collection.document(task.id.uuidString).setData(from: TaskDTO(task, ownerUID: ""), merge: true)
    }

    // ── Supprimer ──────────────────────────────────────
    func delete(id: UUID) async throws {
        try await collection.document(id.uuidString).delete()
    }
}

// ── DTO — objet de transfert pour Firestore ─────────
struct TaskDTO: Codable {
    let id:        String
    let ownerUID:  String
    let title:     String
    let notes:     String
    let isDone:    Bool
    let priority:  Int
    let dueDate:   Date?
    @ServerTimestamp var createdAt: Date?   // auto-généré par Firestore

    init(_ task: Task, ownerUID: String) {
        self.id       = task.id.uuidString
        self.ownerUID = ownerUID
        self.title    = task.title
        self.notes    = task.notes
        self.isDone   = task.isDone
        self.priority = task.priority
        self.dueDate  = task.dueDate
        self.createdAt = nil
    }

    func toDomain() -> Task? {
        guard let uuid = UUID(uuidString: id) else { return nil }
        return Task(id: uuid, title: title, notes: notes,
                     isDone: isDone, priority: priority, dueDate: dueDate)
    }
}

Security Rules

Firestore Rules · Auth guards · Validation

Les Security Rules Firestore sont votre dernière ligne de défense. Elles s'exécutent côté serveur — un utilisateur malveillant qui contournerait votre app ne peut pas accéder aux données d'un autre.

🚨

Règle de départ : commencez toujours par allow read, write: if false; sur toutes les collections, puis ouvrez uniquement ce qui est nécessaire. Ne faites jamais l'inverse (tout ouvrir et restreindre).

firestore.rules — règles complètes production-ready
rules_version = '2';
service cloud.firestore {
  match /databases/(database)/documents {

    // ── Helpers ──────────────────────────────────────
    function isAuthenticated() {
      return request.auth != null;
    }

    function isOwner(uid) {
      return isAuthenticated() && request.auth.uid == uid;
    }

    function isValidTask() {
      let data = request.resource.data;
      return data.title is string
          && data.title.size() > 0
          && data.title.size() <= 200
          && data.isDone is bool
          && data.priority is int
          && data.ownerUID == request.auth.uid;  // owner = user connecté
    }

    function isNotChangingOwner() {
      return !('ownerUID' in request.resource.data.diff(resource.data).affectedKeys());
    }

    // ── Users ─────────────────────────────────────────
    match /users/{uid} {
      // Lecture de son propre profil uniquement
      allow read:   if isOwner(uid);
      // Création lors du premier sign in
      allow create: if isOwner(uid);
      // Mise à jour sauf les champs sensibles
      allow update: if isOwner(uid)
                    && !('createdAt' in request.resource.data.diff(resource.data).affectedKeys());
      allow delete: if false;  // jamais de suppression de profil
    }

    // ── Tasks ─────────────────────────────────────────
    match /tasks/{taskId} {
      // Lire uniquement ses propres tâches
      allow read:   if isAuthenticated()
                    && resource.data.ownerUID == request.auth.uid;
      // Créer une tâche avec validation
      allow create: if isAuthenticated() && isValidTask();
      // Mettre à jour sans changer le propriétaire
      allow update: if isAuthenticated()
                    && resource.data.ownerUID == request.auth.uid
                    && isNotChangingOwner();
      // Supprimer ses propres tâches
      allow delete: if isAuthenticated()
                    && resource.data.ownerUID == request.auth.uid;
    }
  }
}
✅

Tester les rules localement : utilisez l'émulateur Firebase (firebase emulators:start) et le FirebaseEmulatorSuite pour tester vos rules sans toucher à la production. Ajoutez des tests de rules avec @firebase/rules-unit-testing dans votre CI.

import FirebaseAnalytics

// MARK: — LAYER: Infrastructure
protocol AnalyticsServiceProtocol: Sendable {
    func log(_ event: AnalyticsEvent)
    func setUserProperty(_ value: String?, forName name: String)
    func setUserID(_ id: String?)
    func setScreen(name: String, class screenClass: String)
}

final class FirebaseAnalyticsService: AnalyticsServiceProtocol, Sendable {

    func log(_ event: AnalyticsEvent) {
        guard AppConfiguration.analyticsEnabled else { return }
        Analytics.logEvent(event.name, parameters: event.parameters.isEmpty ? nil : event.parameters)
    }

    func setUserProperty(_ value: String?, forName name: String) {
        guard AppConfiguration.analyticsEnabled else { return }
        Analytics.setUserProperty(value, forName: name)
    }

    func setUserID(_ id: String?) {
        guard AppConfiguration.analyticsEnabled else { return }
        Analytics.setUserID(id)
    }

    func setScreen(name: String, class screenClass: String) {
        guard AppConfiguration.analyticsEnabled else { return }
        Analytics.logEvent(AnalyticsEventScreenView, parameters: [
            AnalyticsParameterScreenName:  name,
            AnalyticsParameterScreenClass: screenClass
        ])
    }
}

// Mock pour les tests et le debug
final class MockAnalyticsService: AnalyticsServiceProtocol, Sendable {

        print("[Analytics] \(event.name) \(event.parameters)")
    }
    func setUserProperty(_ v: String?, forName n: String) {}
    func setUserID(_ id: String?) {}
    func setScreen(name: String, class c: String) {}
}

struct ScreenTrackingModifier: ViewModifier {

    let screenClass: String

            container.analytics.setScreen(name: screenName, class: screenClass)

extension View {

Firebase Crashlytics

CrashlyticsService · dSYM · Non-fatal · Keys

Crashlytics capture automatiquement tous les crashes. Mais sa vraie puissance vient des logs, clés et erreurs non-fatales que vous ajoutez manuellement — ils permettent de reconstituer le contexte exact d'un crash.

Upload des dSYM — configuration Xcode obligatoire

Build Phase — Upload dSYM

Target → Build Phases → + New Run Script Phase. Mettez ce script APRÈS "Link Binary with Libraries". Sans ça, les crash reports affichent des adresses mémoire illisibles au lieu des noms de fonctions.

Infrastructure/CrashlyticsService.swift
import FirebaseCrashlytics

// MARK: — LAYER: Infrastructure
protocol CrashlyticsServiceProtocol: Sendable {
    func setUserID(_ id: String)
    func setKey(_ key: CrashlyticsKey, value: some CustomStringConvertible)
    func log(_ message: String)
    func record(error: some Error)
    func crash()  // test crash uniquement
}

// Clés Crashlytics — jamais de string en dur dans le code
enum CrashlyticsKey: String {
    case subscriptionTier   = "subscription_tier"
    case appVersion         = "app_version"
    case currentScreen      = "current_screen"
    case lastAction         = "last_action"
    case taskCount          = "task_count"
    case networkReachable   = "network_reachable"
}

final class FirebaseCrashlyticsService: CrashlyticsServiceProtocol, Sendable {

    func setUserID(_ id: String) {
        Crashlytics.crashlytics().setUserID(id)
    }

    func setKey(_ key: CrashlyticsKey, value: some CustomStringConvertible) {
        Crashlytics.crashlytics().setCustomValue(value.description, forKey: key.rawValue)
    }

    func log(_ message: String) {
        Crashlytics.crashlytics().log(message)
    }

    // Erreur non-fatale — apparaît dans Crashlytics sans crash
    func record(error: some Error) {
        Crashlytics.crashlytics().record(error: error)
    }

    // ⚠ Uniquement pour tester l'intégration en debug
    func crash() {
        #if DEBUG
        fatalError("Test Crashlytics crash")
        #endif
    }
}

// Utilisation dans AppContainer après auth
extension AppContainer {
    func configureCrashlytics(for user: AppUser) {
        crashlytics.setUserID(user.uid)
        crashlytics.setKey(.appVersion, value: Bundle.main.infoDictionary?["CFBundleShortVersionString"] as? String ?? "unknown")
    }
}

Mode Offline

Cache Firestore · Persistance · Sync automatique

Firestore supporte le mode offline natif — les lectures et écritures fonctionnent sans connexion, et la synchronisation se fait automatiquement à la reconnexion. Il faut l'activer explicitement.

Data/Firestore/FirestoreConfiguration.swift
import FirebaseFirestore

enum FirestoreConfiguration {

    static func configure() {
        let settings = FirestoreSettings()

        // Cache local persistant (remplace l'ancien enablePersistence)
        settings.cacheSettings = PersistentCacheSettings(sizeBytes: FirestoreCacheSizeUnlimited)

        // En debug : utiliser l'émulateur local
        #if DEBUG
        if AppConfiguration.useEmulator {
            let db = Firestore.firestore()
            db.useEmulator(withHost: "localhost", port: 8080)
            return  // pas de cache sur l'émulateur
        }
        #endif

        Firestore.firestore().settings = settings
    }
}

Listeners et offline : les addSnapshotListener continuent de fonctionner offline — ils retournent les données du cache local. Quand la connexion revient, Firebase envoie automatiquement un nouveau snapshot avec les données serveur. Votre UI se met à jour sans code supplémentaire.
